# اعتبارسنجی و امنیت

این فصل از مهم‌ترین فصل‌های کل دوره است. اگر PHP بلد باشی ولی اعتبارسنجی و امنیت را جدی نگیری، برنامه‌ات خیلی سریع به منبع باگ، نشت داده، و حمله تبدیل می‌شود.

خبر خوب این است که بخش زیادی از امنیت، از عادت‌های درست می‌آید نه از جادو:

- ورودی را بررسی کن
- خروجی را escape کن
- query را concatenate نکن
- رمز عبور را hash کن
- درخواست‌های حساس را با CSRF token محافظت کن

## هدف این فصل

در پایان این فصل باید بتوانی:

- ورودی‌های کاربر را validate کنی
- از XSS جلوگیری کنی
- CSRF token بسازی و بررسی کنی
- SQL injection را بفهمی و از آن دوری کنی
- مفاهیم امنیتی پایه مثل HTTPS، cookie flags و CSP را بشناسی

## جمله‌ای که باید همیشه یادت بماند

«دادهٔ کاربر دشمن نیست، اما trusted هم نیست.»

یعنی نه باید از کاربر بترسی، نه باید کورکورانه به داده‌اش اعتماد کنی. راه درست این است که داده را در هر مرحله متناسب با context بررسی و کنترل کنی.

در این فصل عمداً چند عادت را بارها تکرار می‌کنیم:

- `htmlspecialchars(..., ENT_QUOTES, 'UTF-8')`
- مقایسه با `===`
- whitelist به‌جای blacklist
- prepared statements به‌جای string concatenation

تکرار این‌ها خسته‌کننده نیست؛ تبدیل کردن‌شان به پیش‌فرض ذهنی است.
