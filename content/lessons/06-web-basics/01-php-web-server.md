# سرور داخلی PHP

برای یادگیری، لازم نیست از همان روز اول Nginx یا Apache راه بیندازی. PHP یک سرور داخلی ساده دارد که برای تمرین، دمو، و پروژه‌های کوچک آموزشی عالی است.

فرمان اجرا:

```bash
php -S localhost:8000
```

اگر فایل `index.php` در پوشهٔ فعلی باشد، با باز کردن `http://localhost:8000` اجرا می‌شود.

اگر بخواهی پوشهٔ مشخصی را به‌عنوان document root بدهی:

```bash
php -S localhost:8000 -t public
```

## یک مثال خیلی ساده

فایل `index.php`:

```php
<?php

declare(strict_types=1);

echo 'سلام از PHP';
```

حالا با باز کردن آدرس بالا، مرورگر این متن را نمایش می‌دهد.

## چرخهٔ request/response یعنی چه؟

هر بار که مرورگر صفحه‌ای را باز می‌کند، تقریباً این اتفاق می‌افتد:

1. مرورگر یک درخواست HTTP می‌فرستد.
2. سرور PHP آن درخواست را دریافت می‌کند.
3. کد PHP اجرا می‌شود.
4. PHP یک پاسخ HTTP برمی‌گرداند.
5. مرورگر پاسخ را نمایش می‌دهد.

این پاسخ می‌تواند HTML، JSON، فایل دانلودی، یا حتی فقط یک کد وضعیت مثل `404` باشد.

## درخواست چه چیزهایی دارد؟

درخواست معمولاً شامل این‌هاست:

- `method` مثل `GET` یا `POST`
- `path` مثل `/contact`
- `query string` مثل `?page=2`
- `headers`
- `body` برای فرم‌ها یا JSON

در PHP خیلی وقت‌ها از متغیرهای superglobal برای دسترسی به این اطلاعات استفاده می‌کنیم:

- `$_SERVER`
- `$_GET`
- `$_POST`
- `$_FILES`
- `$_COOKIE`

## پاسخ چه چیزهایی دارد؟

پاسخ هم معمولاً شامل این‌هاست:

- status code مثل `200` یا `302`
- headers
- body

مثلاً این کد نوع پاسخ را HTML تعیین می‌کند:

```php
<?php

declare(strict_types=1);

header('Content-Type: text/html; charset=UTF-8');

echo '<h1>سلام</h1>';
```

## هر refresh یعنی اجرای دوباره

یکی از مهم‌ترین نکته‌ها برای تازه‌کارها این است:

PHP در مدل سنتی وب، برای هر درخواست از اول اجرا می‌شود. یعنی متغیرهای عادی بین دو refresh حفظ نمی‌شوند.

این کد:

```php
<?php

$count = 0;
$count++;

echo $count;
```

همیشه `1` چاپ می‌کند، چون در هر درخواست از اول شروع شده است.

اگر بخواهی داده بین درخواست‌ها حفظ شود، بعداً از `session`، `cookie`، فایل، یا دیتابیس استفاده می‌کنی.

## تفاوت GET و POST در تصویر کلی

- `GET` بیشتر برای خواندن داده است.
- `POST` بیشتر برای ارسال و تغییر داده است.

فعلاً همین را یادت بماند: هر چیزی که کاربر می‌فرستد، غیرقابل اعتماد است و باید بررسی شود.

## نمونهٔ کوچک از اطلاعات درخواست

```php
<?php

declare(strict_types=1);

header('Content-Type: text/plain; charset=UTF-8');

echo 'Method: ' . $_SERVER['REQUEST_METHOD'] . PHP_EOL;
echo 'URI: ' . $_SERVER['REQUEST_URI'] . PHP_EOL;
echo 'Host: ' . $_SERVER['HTTP_HOST'] . PHP_EOL;
```

## هشدار مهم امنیتی

سرور داخلی PHP برای توسعه و آموزش است، نه برای production. برای سایت واقعی باید از وب‌سرور مناسب، HTTPS، لاگ‌گیری، و تنظیمات امنیتی درست استفاده کنی.

## جمع‌بندی

در این درس دو چیز کلیدی را فهمیدی:

- با `php -S` خیلی سریع می‌توانی محیط تمرین بسازی.
- وب در PHP یعنی دریافت request و تولید response.

در درس بعدی می‌رویم سراغ گرفتن داده از `$_GET` و `$_POST` و این‌که چطور این کار را تمیزتر و امن‌تر انجام بدهیم.
