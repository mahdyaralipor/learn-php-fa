# استقرار و اجرای پروژه

نوشتن کد فقط نصف راه است. بخش مهم دیگر این است که پروژه را در محیط واقعی اجرا کنی، خطاها را درست مدیریت کنی و روندی داشته باشی که هر تغییر با اعتماد بیشتری منتشر شود.

## در این فصل چه یاد می‌گیری؟

- مسیر رسیدن request به PHP-FPM
- الگوی تنظیمات مبتنی بر env
- خاموش کردن نمایش خطا در production
- logging کاربردی
- آشنایی با CI و ایدهٔ GitHub Actions

## هدف این فصل

نمی‌خواهیم همهٔ جزئیات DevOps را پوشش بدهیم. هدف این است که وقتی پروژهٔ PHP را deploy می‌کنی، حداقل‌های مهم را رعایت کرده باشی:

- تنظیمات حساس داخل سورس نباشد
- خطاها به کاربر لو نروند
- لاگ داشته باشی
- یک pipeline ساده برای چک‌کردن سلامت کد داشته باشی
