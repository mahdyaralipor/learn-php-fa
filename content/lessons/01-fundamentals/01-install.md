# نصب و راه‌اندازی PHP

قبل از هر چیز باید PHP را روی سیستم‌ات داشته باشی و بتوانی یک فایل ساده را اجرا کنی.
خبر خوب این است که روی اوبونتو و اکثر توزیع‌های لینوکسی، نصب PHP معمولاً ساده است.

## PHP چیست؟

PHP یک زبان برنامه‌نویسی سمت سرور است.
یعنی کدی که می‌نویسی روی سرور اجرا می‌شود و خروجی‌اش — مثلاً HTML یا JSON — به مرورگر یا ترمینال می‌رسد.

در این دوره با **PHP 8.2+** کار می‌کنیم.
نسخه‌های جدیدتر type hint بهتر، enum، readonly property و امکانات مدرن‌تری دارند.

## نصب روی اوبونتو / دبیان

```bash
sudo apt update
sudo apt install php-cli php-mbstring php-xml php-curl
```

پکیج `php-cli` برای اجرای فایل‌ها از ترمینال کافی است.
بقیهٔ پکیج‌ها را از همین ابتدا نصب می‌کنیم تا در فصل‌های بعدی گیر نکنی.

## بررسی نسخه

```bash
php -v
```

خروجی شبیه این باید ببینی:

```text
PHP 8.2.x (cli) ...
```

اگر نسخه زیر ۸.۲ بود، بهتر است PHP را به‌روز کنی.

## اولین اجرا بدون فایل

می‌توانی مستقیم از ترمینال کد PHP اجرا کنی:

```bash
php -r 'echo "سلام از PHP" . PHP_EOL;'
```

`PHP_EOL` یعنی «پایان خط» مناسب سیستم‌عامل.
در لینوکس معمولاً `\n` است.

## ساخت پوشهٔ تمرین

```bash
mkdir -p ~/php-course
cd ~/php-course
```

از همین پوشه برای تمرین‌های فصل اول استفاده می‌کنیم.

## اجرای فایل PHP

یک فایل به نام `info.php` بساز:

```php
<?php

echo "PHP آماده است." . PHP_EOL;
```

اجرا:

```bash
php info.php
```

خروجی:

```text
PHP آماده است.
```

## تگ باز PHP

هر فایل PHP معمولاً با `<?php` شروع می‌شود.
در فایل‌های خالص PHP، تگ بسته `?>` لازم نیست و بهتر است حذفش کنی تا فاصلهٔ اضافهٔ بعد از آن باعث خطای header نشود.

## تفاوت CLI و وب

دو روش اصلی برای اجرای PHP داری:

| روش | کاربرد |
|-----|--------|
| `php file.php` | تمرین، اسکریپت، ابزار خط فرمان |
| سرور وب (مثل `php -S`) | صفحات HTML، فرم، API |

در فصل مبانی بیشتر با **CLI** کار می‌کنیم چون سریع‌تر است و برای یادگیری مفاهیم پایه کافی‌تر.

## سرور داخلی PHP (اختیاری در این مرحله)

اگر خواستی فایل را در مرورگر ببینی:

```bash
php -S localhost:8000
```

بعد `http://localhost:8000/info.php` را باز کن.
این موضوع را در فصل وب عمیق‌تر بررسی می‌کنیم.

## `php.ini` چیست؟

تنظیمات کلی PHP در فایل `php.ini` است.
برای دیدن مسیر فایل فعال:

```bash
php --ini
```

فعلاً نیازی به تغییر تنظیمات نیست، اما بدان کجا قرار دارد.

## ابزارهای مفید

```bash
# نمایش اطلاعات کامل PHP
php -i

# بررسی سینتکس فایل بدون اجرا
php -l info.php
```

`php -l` برای پیدا کردن خطای نگارشی قبل از اجرا خیلی کاربردی است.

## اشتباهات رایج

### 1. فراموش کردن `php` در ابتدای دستور

```bash
info.php   # اشتباه
php info.php  # درست
```

### 2. نبودن `<?php` در ابتدای فایل

بدون تگ باز، PHP کد را به‌عنوان متن خام چاپ می‌کند یا خطا می‌دهد.

### 3. اجرا از پوشهٔ اشتباه

اگر `php info.php` خطای «فایل پیدا نشد» داد، اول `pwd` و `ls` بزن و مطمئن شو در پوشهٔ درست هستی.

### 4. نسخهٔ قدیمی PHP

بعضی آموزش‌های قدیمی با PHP 5 یا 7 نوشته شده‌اند.
اگر مثالی با رفتار عجیب مواجه شدی، اول نسخه را چک کن.

## تمرین

1. PHP را نصب کن و `php -v` را اجرا کن؛ نسخه را یادداشت کن.
2. پوشهٔ `~/php-course` بساز و داخلش فایل `hello.php` با یک `echo` بساز.
3. همان فایل را با `php -l` و بعد `php hello.php` اجرا کن.
4. با `php -r` یک جملهٔ فارسی چاپ کن.
5. خروجی `php --ini` را ببین و مسیر `php.ini` را پیدا کن.

## جمع‌بندی

در این درس PHP را نصب کردی، نسخه را بررسی کردی و اولین فایل را از ترمینال اجرا کردی.
از درس بعدی سراغ ساختار واقعی یک برنامهٔ PHP و تفاوت اجرای CLI با وب می‌رویم.
