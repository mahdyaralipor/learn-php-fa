# سلام دنیا

اولین برنامهٔ هر زبان معمولاً یک جملهٔ ساده چاپ می‌کند.
در PHP هم مسیر از همین‌جا شروع می‌شود، اما همزمان چند عادت مهم را از روز اول یاد می‌گیری.

## ساده‌ترین برنامه

فایل `hello.php`:

```php
<?php

echo "سلام دنیا";
```

اجرا:

```bash
php hello.php
```

خروجی:

```text
سلام دنیا
```

## `echo` چه می‌کند؟

`echo` یکی از راه‌های چاپ خروجی است.
می‌تواند رشته، عدد و حتی نتیجهٔ عبارت را نمایش دهد:

```php
<?php

echo "سلام" . PHP_EOL;
echo 42 . PHP_EOL;
echo 10 + 5 . PHP_EOL;
```

## `print` هم هست

```php
<?php

print "با print هم می‌شود" . PHP_EOL;
```

تفاوت عملی برای شروع کم است؛ `echo` کمی سریع‌تر است و بیشتر استفاده می‌شود.

## کامنت‌ها

```php
<?php

// این یک خط کامنت است

/*
  این کامنت
  چندخطی است
*/

# این هم یک خط کامنت است
```

کامنت برای توضیح منطق به انسان است؛ PHP آن را اجرا نمی‌کند.

## ساختار یک فایل PHP

```php
<?php

// 1. تعریف داده
$name = "رضا";

// 2. پردازش
$greeting = "سلام، {$name}";

// 3. خروجی
echo $greeting . PHP_EOL;
```

خروجی:

```text
سلام، رضا
```

## `declare(strict_types=1)`

در PHP مدرن معمولاً اول فایل این خط را می‌بینیم:

```php
<?php

declare(strict_types=1);

echo "حالت strict فعال است" . PHP_EOL;
```

این دستور به PHP می‌گوید در همان فایل، تبدیل نوع خودکار در type hintها سخت‌گیرانه‌تر باشد.
فعلاً جزئیاتش را نمی‌خواهیم؛ فقط بدان جای درستش **اول فایل، بعد از `<?php`** است.

## خروجی با `var_dump`

برای یادگیری، `var_dump` از `echo` مفیدتر است چون **نوع** و **مقدار** را با هم نشان می‌دهد:

```php
<?php

$name = "سارا";
$age = 25;

var_dump($name);
var_dump($age);
```

خروجی:

```text
string(8) "سارا"
int(25)
```

## چند خط خروجی

```php
<?php

echo "خط اول" . PHP_EOL;
echo "خط دوم" . PHP_EOL;
echo "خط سوم" . PHP_EOL;
```

یا:

```php
<?php

echo <<<TEXT
خط اول
خط دوم
خط سوم
TEXT;
```

## اجرای وب در یک نگاه

اگر فایل را با سرور داخلی اجرا کنی:

```bash
php -S localhost:8000
```

و `hello.php` را در مرورگر باز کنی، همان متن را می‌بینی.
تفاوت این است که خروجی به مرورگر می‌رود نه ترمینال.

## اشتباهات رایج

### 1. فراموش کردن سمی‌کالن

```php
echo "سلام"  // خطای parse
```

### 2. استفاده از کوتیشن اشتباه

```php
echo 'سلام، {$name}'; // $name جایگزین نمی‌شود
echo "سلام، {$name}"; // درست
```

### 3. فاصله قبل از `<?php`

در فایل‌های وب، هر خروجی قبل از `header()` مشکل‌ساز است.
بنابراین قبل از `<?php` چیزی ننویس.

### 4. بستن تگ `?>` و گذاشتن خط خالی

```php
<?php
echo "سلام";
?>

```

آن خط خالی بعد از `?>` ممکن است باعث «headers already sent» شود.
در فایل خالص PHP تگ بسته را حذف کن.

## تمرین

1. برنامه‌ای بنویس که نام خودت را در سه خط جدا چاپ کند.
2. همان برنامه را با `var_dump` بازنویسی کن و تفاوت خروجی را ببین.
3. یک کامنت چندخطی اضافه کن که توضیح بدهد برنامه چه می‌کند.
4. فایل را با `php -l` بررسی کن؛ عمداً یک سمی‌کالن حذف کن و خطا را ببین.
5. فایل را با `php -S` اجرا کن و در مرورگر باز کن.

## جمع‌بندی

یاد گرفتی `echo` و `print` خروجی می‌دهند، کامنت‌ها توضیح هستند نه کد اجرایی، و `var_dump` برای فهمیدن نوع داده عالی است.
در درس بعدی سراغ متغیرها می‌رویم — جایی که داده را نام‌گذاری و نگه می‌داریم.
