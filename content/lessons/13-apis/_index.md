# ساخت API در PHP

تا اینجا بیشتر خروجی ما HTML بوده است. اما همهٔ برنامه‌ها قرار نیست صفحهٔ وب سنتی برگردانند. خیلی وقت‌ها کلاینت ما می‌تواند یک اپ موبایل، یک SPA جاوااسکریپتی، یا حتی یک سرویس دیگر باشد.

در اینجا API اهمیت پیدا می‌کند.

## در این فصل چه یاد می‌گیری؟

- مفهوم REST و resource-based thinking
- کار با JSON در PHP
- status codeهای مهم API
- احراز هویت ساده با token
- pagination
- ساختن shape ثابت برای خطاها

## هدف این فصل

ما نمی‌خواهیم وارد بحث‌های خیلی سنگین distributed systems شویم. هدف این است که بتوانی یک API تمیز، قابل‌فهم و نسبتاً یکدست برای پروژه‌های PHP بسازی.

## خروجی عملی

در پایان این فصل باید بتوانی:

- response JSON برگردانی
- body JSON را parse کنی
- status code درست انتخاب کنی
- خطاها را ساختارمند برگردانی
- برای لیست‌ها pagination ساده بگذاری
