# متغیرها در PHP

تو PHP هر متغیر با `$` شروع می‌شه. برخلاف خیلی زبون‌ها، نیازی نیست نوع داده رو صریح اعلام کنی — PHP خودش نوع رو از روی مقدار تشخیص می‌ده (Dynamic Typing).

```php
<?php

$name = "Mahdyar";
$age = 28;
$height = 1.78;
$isDeveloper = true;

echo $name;
```

## انواع داده‌ی اصلی

```php
<?php

$string  = "متن";           // string
$integer = 42;               // int
$float   = 3.14;             // float
$boolean = true;             // bool
$array   = [1, 2, 3];        // array
$null    = null;             // null
```

برای دیدن نوع دقیق یه متغیر:

```php
<?php

$x = 42;
var_dump($x);        // int(42)
echo gettype($x);    // integer
```

## نکته‌ی مهم: Type Juggling

این جاییه که خیلی از مبتدی‌ها (و حتی دولوپرهای باتجربه) اشتباه می‌کنن. PHP بین انواع داده تبدیل خودکار انجام می‌ده:

```php
<?php

$result = "5" + 3;      // 8  (رشته به عدد تبدیل شد)
$result = "5" . 3;      // "53" (نقطه یعنی الحاق رشته)
$result = "abc" + 3;    // خطا در PHP 8 (قبلاً 3 برمی‌گردوند!)
```

`.` عملگر الحاق رشته‌ست (concatenation)، نه `+`. این یکی از رایج‌ترین منابع باگ برای کسایی که از زبون‌های دیگه میان.

## مقایسه: `==` در برابر `===`

این یکی حیاتیه، چون تو کد واقعی باگ‌های عجیب می‌سازه اگه ندونیش:

```php
<?php

var_dump(0 == "abc");    // PHP 8: false (در PHP 7 این true بود!)
var_dump("1" == "01");   // true (هر دو به عدد تبدیل می‌شن)
var_dump("1" === "01");  // false (نوع و مقدار دقیق چک می‌شه)
var_dump(1 == "1");      // true
var_dump(1 === "1");     // false
```

**قانون طلایی: همیشه `===` استفاده کن مگه اینکه دلیل مشخصی برای `==` داشته باشی.** این یکی از عاداتیه که یه دولوپر حرفه‌ای رو از مبتدی جدا می‌کنه.

## رشته‌های عددی و مقایسه (نکته‌ی پیشرفته)

وقتی `==` بین دو رشته‌ی عددی (numeric string) استفاده بشه، PHP اونا رو به عدد تبدیل و عددی مقایسه می‌کنه:

```php
<?php

var_dump("10" == "1e1");    // true — چون "1e1" یعنی 1×10¹ = 10
var_dump("10" === "1e1");   // false — با === مقایسه‌ی رشته‌ای دقیقه، نه عددی

var_dump(null == false);    // true — null به بولین تبدیل می‌شه: false
var_dump(null === false);   // false — نوع‌ها فرق دارن (NULL در برابر boolean)
```

**نکته‌ی عملی:** این رفتار مخصوصاً وقتی داری ورودی کاربر (که همیشه string‌ه) رو مقایسه می‌کنی خطرناکه. با `==` ممکنه مقادیر غیرمنتظره برابر تشخیص داده بشن؛ با `===` این ریسک از بین می‌ره.

## تمرین

این کد رو بدون اجرا کردن پیش‌بینی کن، بعد اجراش کن و نتیجه رو با حدست مقایسه کن:

```php
<?php

var_dump("100" == "1e2");
var_dump("abc" == 0);
var_dump([] == false);
var_dump([] === false);
```
