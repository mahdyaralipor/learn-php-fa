# الگوها و بهینه‌سازی عملکرد

وقتی پروژه کمی بزرگ‌تر می‌شود، دو وسوسه همزمان سراغ توسعه‌دهنده می‌آید:

- از هر pattern معروفی استفاده کند چون «حرفه‌ای» به نظر می‌رسد
- برای performance دست به بهینه‌سازی‌های زودهنگام بزند

هر دو می‌توانند خطرناک باشند.

## در این فصل چه یاد می‌گیری؟

- چند pattern رایج و کاربردی
- چرا بعضی patternها باید با احتیاط استفاده شوند
- ایدهٔ cache در PHP
- آشنایی با OPcache
- اندازه‌گیری سادهٔ زمان اجرا
- توجه به N+1 و هزینه‌های پنهان

## هدف این فصل

هدف ما حفظ‌کردن کتاب patternها نیست. می‌خواهیم بفهمیم:

- pattern چه مشکلی را حل می‌کند
- چه زمانی over-engineering می‌شود
- performance را چطور با اندازه‌گیری واقعی ببینیم، نه با حدس
