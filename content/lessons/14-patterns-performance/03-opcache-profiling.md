# OPcache و اندازه‌گیری عملکرد

Performance فقط به الگوریتم کد تو محدود نیست. در PHP، یکی از مهم‌ترین ابزارهای پایه `OPcache` است.

## OPcache چیست؟

PHP قبل از اجرای فایل‌ها، آن‌ها را parse و compile می‌کند. OPcache نتیجهٔ compiled scriptها را cache می‌کند تا درخواست‌های بعدی سریع‌تر اجرا شوند.

به زبان ساده:

- بدون OPcache، هر بار PHP باید زحمت بیشتری بکشد
- با OPcache، بخشی از این هزینه کم می‌شود

## چرا در production مهم است؟

در بیشتر پروژه‌های واقعی، فعال بودن OPcache تقریباً بدیهی است. اگر خاموش باشد، بخشی از performance رایگان را از دست داده‌ای.

## microtime و اندازه‌گیری ساده

قبل از بهینه‌سازی باید اندازه بگیری.

```php
<?php

declare(strict_types=1);

$start = microtime(true);

$posts = $postRepository->latest();

$duration = microtime(true) - $start;

error_log('latest() took ' . number_format($duration, 4) . ' seconds');
```

در PHP 8.2 حتی می‌توانی از `hrtime()` هم برای اندازه‌گیری دقیق‌تر استفاده کنی.

## N+1 یعنی چه؟

یکی از مشکلات رایج در performance این است که به‌جای یک query معقول، ناخواسته تعداد زیادی query بزنی.

مثلاً:

1. همهٔ پست‌ها را می‌گیری
2. برای هر پست، جداگانه نویسنده را query می‌کنی

اگر 100 پست باشد، عملاً 101 query زده‌ای. این همان بوی N+1 است.

## مثال ذهنی

بد:

- یک query برای همهٔ پست‌ها
- یک query برای هر پست جهت گرفتن author

بهتر:

- join مناسب
- یا preload کردن دادهٔ مرتبط

## اصل مهم: اول پیدا کن، بعد بهینه کن

خیلی از optimizeها بر اساس حدس انجام می‌شوند. این کار معمولاً انرژی را جای غلط مصرف می‌کند.

رویکرد بهتر:

1. سنجه داشته باش
2. bottleneck را پیدا کن
3. تغییر کوچک بده
4. دوباره اندازه بگیر

## چیزهایی که باید زیر نظر بگیری

- تعداد queryها
- زمان queryها
- تعداد file includeهای غیرضروری
- loopهای بزرگ
- فراخوانی API بیرونی
- renderهای تکراری

## تمرین

1. با `microtime(true)` مدت اجرای یک متد repository فرضی را اندازه بگیر.
2. یک مثال ذهنی از N+1 در صفحهٔ لیست سفارش‌ها بزن.
3. توضیح بده چرا فعال بودن OPcache در production مهم است.

## جمع‌بندی

برای performance خوب باید هم از ابزارهای پایه مثل OPcache استفاده کنی، هم با اندازه‌گیری واقعی کار کنی. `microtime()` و آگاهی از بوی N+1، دو قدم ساده اما بسیار مؤثر برای شروع بهینه‌سازی آگاهانه‌اند.
