# نشست و کوکی

HTTP ذاتاً stateless است؛ یعنی هر درخواست به‌تنهایی می‌آید و سرور به‌صورت پیش‌فرض حافظه‌ای از درخواست قبلی ندارد.

اما اپلیکیشن واقعی نیاز دارد بداند:

- کاربر login کرده یا نه
- پیام موفقیت قبلی چه بوده
- سبد خرید چیست

اینجاست که `session` و `cookie` وارد می‌شوند.

## در این فصل چه یاد می‌گیری؟

- شروع و استفاده از `session`
- تنظیم امن cookie
- پیاده‌سازی flash message
- اسکلت سادهٔ login/logout بدون دیتابیس

## ذهنیت مهم

state یعنی قدرت، و هر جا قدرت هست باید امنیت هم باشد. پس در این فصل روی این پیش‌فرض‌ها حساس می‌مانیم:

- `session_start()` را درست و به‌موقع بزن
- بعد از login از `session_regenerate_id()` استفاده کن
- cookieهای حساس را با `Secure`، `HttpOnly` و `SameSite` تنظیم کن
- دادهٔ session را trusted مطلق فرض نکن؛ فقط managed فرض کن
